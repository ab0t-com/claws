### Network Isolation Between Instances
Each instance gets its own Docker bridge network (`openclaw-team-sarah_default`, etc.). Instances **cannot** reach each other's containers directly. Communication is only through the shared filesystem (workspace, tasks, output dirs).

### What Admin Cannot Currently Control
1. Cannot restrict an instance to loopback-only binding after creation
2. Cannot set memory/CPU limits per instance
3. Cannot enforce a tool policy across all instances
4. Cannot block specific channels for specific instances
5. Cannot set a global security baseline that overrides per-instance config
6. Cannot restrict which users can run claws commands
7. Cannot audit who created/modified/deleted instances
8. Cannot rotate gateway tokens without manual env file editing
9. Cannot enforce sandbox mode globally
10. Cannot restrict image versions (any OPENCLAW_IMAGE value is accepted)
