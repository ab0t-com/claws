$---- [x] 01-file-locking

## 01 — Add File Locking to Registry and Config Writes

**Severity:** CRITICAL (P0)
**Problem:** All state files (.port-registry, .group.json, .storage.json, instance.env) are read-modify-written without `flock()`. Two concurrent `claws create` calls will race and corrupt the port registry.

**Files & Lines:**
- `registry.go:73-84` — `registerPort()` opens file for append with no lock
- `registry.go:86-102` — `unregisterPort()` reads then writes with no lock
- `registry.go:57-71` — `nextIndex()` reads registry but index can be claimed by another process before write
- `group.go:93-99` — `WriteGroupConfig()` writes `.group.json` without lock
- `storage.go:35-38` — `writeStorageConfig()` writes `.storage.json` without lock
- `shared.go:67-82` — `setSharedFlag()` read-modify-writes `instance.env` without lock

**Implementation:**
1. Create `flock.go` with a `withFileLock(lockPath string, fn func() error) error` helper using `syscall.Flock(fd, LOCK_EX)` with defer unlock
2. Add a `withRegistryLock(paths Paths, fn func() error) error` that locks `<root>/.port-registry.lock`
3. Wrap `registerPort`, `unregisterPort`, `nextIndex` call sites in `withRegistryLock`
4. Add per-group lock for `.group.json` writes
5. Add per-instance lock for `instance.env` modifications

**Tests:**
- Unit test: `TestFileLock_Exclusive` — two goroutines trying to lock the same file, second blocks
- Unit test: `TestRegistryLock_ConcurrentRegister` — parallel `registerPort` calls don't corrupt registry
- Integration test: `TestIntegration_ConcurrentCreate` — two `claws create` via goroutines both succeed with different ports

## 02 — Fix instance.env File Permissions (0644 → 0600)

**Severity:** CRITICAL (P0)
**Problem:** `instance.env` contains `OPENCLAW_GATEWAY_TOKEN` (256-bit auth secret) but is written with world-readable 0644. Should be 0600.

**Files & Lines:**
- `commands.go:181` — `os.WriteFile(envFile, []byte(envContent), 0644)` — CREATE path
- `commands.go:816` — `os.WriteFile(envFile, ...data..., 0644)` — `updateEnvValue()`
- `shared.go:40` — `os.OpenFile(envFile, os.O_APPEND|os.O_WRONLY, 0644)` — `writeSharedFlags()`
- `shared.go:81` — `os.WriteFile(envFile, ...data..., 0644)` — `setSharedFlag()`
- `registry.go:22` — `os.WriteFile(paths.PortRegistry, nil, 0644)` — registry also should be 0600
- `registry.go:77` — `os.OpenFile(paths.PortRegistry, ..., 0644)` — append mode
- `registry.go:101` — `os.WriteFile(paths.PortRegistry, ..., 0644)` — rewrite

**Implementation:**
1. Define `const credentialFileMode = 0600` in `config.go`
2. Replace all `0644` on `instance.env` and `.port-registry` writes with `credentialFileMode`
3. Leave non-sensitive files (override YAML, openclaw.json, .group.json) at 0644

**Tests:**
- Unit test: `TestEnvFilePermissions` — after `cmdCreate`, stat instance.env and verify mode is 0600
- Unit test: `TestRegistryFilePermissions` — after `registerPort`, verify .port-registry is 0600
- Integration test: `TestIntegration_CreateEnvPermissions` — run `claws create`, check file mode

**Files & Lines:**

4. Update help text in `main.go` to reflect: `"storage sync [--dry-run] [--mirror]  Sync to S3 (default: additive copy)"`
