The Problem

Running multiple AI agents on one server shouldn't require a PhD in DevOps.

  • ✕ Port conflicts and credential leaks between instances
  • ✕ Manual Docker compose wrangling for every new agent
  • ✕ No visibility into health, RAM, or uptime across the fleet
  • ✕ Workspace sharing and collaboration is an afterthought
  • ✕ Backup means copying directories and hoping for the best
The Solution

claws makes multi-agent orchestration feel like managing one.

  • ✓ Automatic port allocation and complete namespace isolation
  • ✓ One command to create, start, stop, or remove any instance
  • ✓ Live dashboard with health probes, RAM, and uptime per agent
  • ✓ Group instances with shared skills, workspace, and hooks
  • ✓ S3 backup, FUSE mounting, and automated cron sync built in
Capabilities

Everything you need to run an agent fleet.

From single-instance management to multi-agent team orchestration with durable storage and observability.

01
⚡

Instant Provisioning

Create a fully isolated agent instance in under 5 seconds. Ports, tokens, config, workspace — all handled.

Each instance gets its own openclaw.json, credentials directory, workspace, and Docker Compose project. Port allocation uses a deterministic registry: base 18789 + (index × 100), with automatic gap reuse when instances are removed.
02
🛡️

Complete Isolation

Credentials never cross instance boundaries. Each agent runs in its own Docker namespace with dedicated volumes.

Isolation is achieved through Docker Compose project naming, per-instance env files, and separate volume mounts. Shared resources (skills, workspace, hooks) are opt-in and use Docker bind mounts with configurable read-only/read-write access.
03
📊

Live Dashboard

Real-time fleet overview with health probes, RAM usage, uptime, and failing component detection.

The dashboard refreshes at configurable intervals, probing both /healthz (liveness) and /readyz (readiness) endpoints. Verdicts: healthy, degraded, stopped, or down — with specific failing component names surfaced from the readiness response.
04
👥

Group Orchestration

Organize agents into teams with shared resources, config inheritance, and manager/worker roles.

Groups provide a namespace: claws create backend/sarah. Config inherits through 4 layers: global defaults → group defaults → template → instance skeleton. Grouped instances share resources by default — opt out with --no-shared-workspace.
05
☁️

S3 Durability

Backup to S3 with versioning, automated cron sync, FUSE mounting, and workspace migration.

Storage setup creates the S3 bucket, enables versioning, blocks public access, and configures rclone — all in one command. Sync excludes credentials and sessions. FUSE mount via mountpoint-s3 enables direct filesystem access to S3-backed workspaces.
06
🔒

Reverse Proxy + HTTPS

Expose agents through Caddy with automatic Let's Encrypt certificates. Path or subdomain routing.

Two routing modes: path-based (domain.com/sarah/) for simple setups, or subdomain (sarah.domain.com) for clean URLs. Caddy handles certificate provisioning and renewal automatically. One command sets up and configures everything.
Architecture

Three layers. Zero magic.

CLI claws binary ▼

Single static Go binary (~8MB) with zero runtime dependencies. Parses commands, manages the port registry, generates compose overrides, merges configs, and orchestrates the lifecycle of every instance. All state lives in ~/.openclaw/ as flat files — inspectable, grep-able, and version-controllable.

COMPOSE Docker Compose substrate ▼

A shared docker-compose.yml template defines the gateway and CLI services. Per-instance instance.env files inject ports, tokens, and paths. Generated docker-compose.override.yml files add shared resource mounts, role-specific volumes, and environment overrides — never hand-edited.

GATEWAY OpenClaw gateway process ▼

Each instance runs a single long-lived OpenClaw gateway process inside its container. The gateway owns session state, channel integrations (WhatsApp, Telegram, Slack, Discord, and 30+ more), agent runtime, and the embedded workspace. It exposes /healthz and /readyz endpoints for claws to probe.

Getting Started

From zero to a running agent fleet in four steps.

1

Create an instance

Name it, and claws handles the rest — ports, tokens, config, directory structure.

terminal
$ claws create sarah
==> Creating instance 'sarah'...
  Index:     0
  Gateway:   :18789
  Directory: ~/.openclaw/sarah
==> Instance 'sarah' created.
2

Add authentication

Wire up an AI provider — OAuth for Codex or a raw API key for OpenAI, Anthropic, or Gemini.

terminal
$ claws auth sarah apikey anthropic sk-ant-...
==> Adding anthropic API key to 'sarah'...
==> Restarting gateway...
==> Auth complete for 'sarah'.
3

Start the instance

The gateway boots, claws waits for the health check, and reports readiness.

terminal
$ claws start sarah
==> Starting instance 'sarah'...
==> Waiting for health...
==> Instance 'sarah' is healthy on :18789
4

Scale to a fleet

Create more instances, organize into groups, monitor from the dashboard.

terminal
$ claws group create backend
==> Group 'backend' created.
$ claws create backend/bob --from=sarah
==> Instance 'backend/bob' created.
  (config copied from sarah)
$ claws dashboard

Durability

Your agents' work, backed up and durable.

S3 integration built in — not bolted on. From manual backup to fully automated cloud-synced workspaces.

📦

Tarball Backup

Instant tarball backups with timestamps. Restore to any name with automatic port re-registration. Simple, portable, reliable.

🔄

rclone Sync

Sync workspaces to S3 with smart exclusions — credentials and sessions never leave the server. Supports dry-run for safe previews.

📂

FUSE Mount

Mount S3 directly as a local filesystem via mountpoint-s3. Migrate instance workspaces to S3-backed storage with zero downtime to configuration.

⏰

Cron Automation

Enable automatic sync on an hourly, daily, or custom schedule. One command to enable, one to disable. Logs to /var/log for auditability.

🛡️

Versioned Buckets

Storage setup enables S3 versioning and blocks public access by default. Your agents' work is recoverable even after accidental overwrites.

🔀

Live Migration

Move a running instance's workspace from local disk to S3. claws stops, copies, re-points, and restarts — cleanly and atomically.
