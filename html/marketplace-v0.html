How it works

A template is a JSON profile describing the desired state — runtime, team, agents, channels, policy. claws apply reconciles your host to match. Same template works for one-shot EC2 provisioning and for interactive laptop setup.

// solo-telegram-coder/profile.json { "apiVersion": "claws.ab0t.com/v1", "kind": "Profile", "metadata": { "name": "solo-telegram-coder", "version": "1.2.0" }, "runtime": { "name": "openclaw", "image": "openclaw:local" }, "policy": { "loopbackOnly": true, "dmDefault": "pairing" }, "team": { "name": "default" }, "agents": [ { "name": "agent-1", "auth": { "preferred": "codex" }, "channels": [ { "type": "telegram", "tokenFrom": { "env": "TELEGRAM_BOT_TOKEN" } } ] } ], "postSetup": ["audit", "verify --all", "start --all"] }
Secrets stay local. Profiles reference tokenFrom.env, tokenFrom.file, or tokenFrom.command — never inline. Publishable templates contain no credentials.

Same templates, two journeys

Whatever runs in cloud-init also runs on your laptop. The difference is whether you ship secrets ahead of time or get prompted for them.

Zero-touch (cloud) EC2-ready

For cloud-init, Dockerfile, k8s jobs, CI smoke tests. No prompts. Exits 0 only if everything verified healthy.

# EC2 user_data curl -fsSL https://raw.githubusercontent.com/ab0t-com/claws/main/scripts/install.sh | sh claws apply --template=solo-telegram-coder \ --secrets-dir=/etc/claws/secrets \ --yes --verify
Interactive (laptop) Smart defaults

For developers trying claws for the first time. Prompts only for what only you can answer — the OAuth click and the bot token.

# Just run it $ claws quickstart # or explicit template $ claws quickstart --template=team-research

Submit your own template

Built a useful claws configuration? Share it. We curate community submissions and promote good ones to verified status. Templates are MIT-licensed by default.

In 3 steps

  1. Author your profile.json against the v1 schema.
  2. Add a README.md (use case, prereqs) and secrets.example.env listing required env vars.
  3. Open a PR to ab0t-com/claws-templates. We validate against the schema, dogfood it on a fresh EC2, then merge.
