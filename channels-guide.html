Channel Setup Guide

Step-by-step instructions for connecting your AI agents to every supported messaging platform.

  • ✈ Telegram
  • ⚛ Discord
  • ▦ Slack
  • ☎ WhatsApp
  • ⚠ Signal

Telegram ~2 min

Create a bot via @BotFather, paste the token, done.

  1. Open @BotFather in Telegram

    Go to t.me/BotFather or search "BotFather" in the Telegram app.

  2. Create a new bot

    Send /newbot. BotFather will ask for a display name (anything you want) and a username (must end in bot, e.g., my_team_bot).

  3. Copy the token

    BotFather replies with a token like 7123456789:AAH.... Copy the entire line.

  4. Add to your agent
    # During setup (interactive prompt)
    clawctl setup

    # Or add to an existing agent
    clawctl channel add my-agent telegram --token=7123456789:AAH...

    # Or inline with create
    clawctl create alice --telegram=7123456789:AAH...
  5. Approve the pairing

    Message the bot in Telegram. It will show a pairing code. Enter it:

    clawctl approve my-agent telegram CODE123
Tip: You can customize the bot's profile picture and description in @BotFather with /setuserpic and /setdescription.

Discord ~5 min

Create an application in the Discord Developer Portal, add a bot, invite it to your server.

  1. Go to the Developer Portal

    Visit discord.com/developers/applications and log in.

  2. Create a new application

    Click "New Application". Give it a name (this appears in the sidebar). Click "Create".

  3. Add a bot

    Go to the "Bot" tab in the left sidebar. Click "Add Bot" → "Yes, do it!".

  4. Enable message content intent

    Still on the Bot tab, scroll down to "Privileged Gateway Intents". Enable "Message Content Intent". Save.

  5. Copy the token

    Click "Reset Token" (or "Copy" if visible). This is your bot token. Save it somewhere safe — you can only see it once.

  6. Invite the bot to your server

    Go to the "OAuth2" tab → "URL Generator". Check these scopes:

    • bot
    • applications.commands

    Under "Bot Permissions", select: Send Messages, Read Message History, Add Reactions, Use Slash Commands.

    Copy the generated URL at the bottom and open it in your browser. Select your server and authorize.

  7. Add to your agent
    clawctl channel add my-agent discord --token=MTIz...

    # Or inline with create
    clawctl create alice --discord=MTIz...
  8. Approve the pairing

    DM the bot on Discord. Enter the pairing code:

    clawctl approve my-agent discord CODE123
Important: The "Message Content Intent" is required for the bot to read messages. Without it, the bot can see that a message was sent but not its content.

Slack ~10 min

Slack requires creating an app with two tokens (bot + app-level) and configuring socket mode.

  1. Go to the Slack API portal

    Visit api.slack.com/apps and sign in to your workspace.

  2. Create a new app

    Click "Create New App" → "From scratch". Enter a name and select your workspace.

  3. Enable Socket Mode

    In the left sidebar, go to "Socket Mode". Toggle it ON. When prompted, create an app-level token with the connections:write scope. Name it anything (e.g., "clawctl"). Copy the xapp-... token.

  4. Subscribe to events

    Go to "Event Subscriptions". Toggle ON. Under "Subscribe to bot events", add:

    • message.im (direct messages to bot)
    • message.channels (messages in public channels)
    • app_mention (when someone @mentions the bot)

    Click "Save Changes".

  5. Add bot scopes

    Go to "OAuth & Permissions". Under "Bot Token Scopes", add:

    • chat:write
    • channels:history
    • im:history
    • im:read
    • reactions:write
    • users:read
  6. Install to workspace

    Click "Install to Workspace" at the top of the OAuth page. Authorize. Copy the Bot User OAuth Token (xoxb-...).

  7. Add to your agent
    clawctl channel add my-agent slack \
      --bot-token=xoxb-... \
      --app-token=xapp-...
  8. Invite the bot to a channel

    In Slack, go to the channel where you want the bot. Type /invite @your-bot-name.

Tip: You need two tokens for Slack. The xoxb- bot token handles API calls. The xapp- app-level token handles the WebSocket connection (Socket Mode).

WhatsApp ~5 min

WhatsApp uses QR code login — no developer portal needed. You'll need a phone number dedicated to the bot.

  1. Get a phone number for the bot

    You need a phone number that is not already registered with WhatsApp. A cheap prepaid SIM or a VoIP number works. You'll receive one SMS to verify it.

  2. Start the channel setup
    clawctl channel add my-agent whatsapp

    This starts an interactive flow that will display a QR code in your terminal.

  3. Scan the QR code

    Open WhatsApp on the phone with your bot's number. Go to Settings → Linked Devices → Link a Device. Scan the QR code shown in your terminal.

  4. Wait for connection

    The terminal will show connection progress. Once linked, the agent can receive and respond to WhatsApp messages.

  5. Approve the pairing

    Send a message to the bot's WhatsApp number from your personal phone. Enter the pairing code:

    clawctl approve my-agent whatsapp CODE123
Important: WhatsApp's default DM policy is allowlist (stricter than pairing). You must explicitly add approved contacts: clawctl channel allow my-agent whatsapp +1234567890
Tip: The QR session expires after ~60 seconds. If it times out, run the command again.

Signal ~5 min

Signal uses a phone number. You'll need signal-cli or a linked device setup.

  1. Get a phone number for the bot

    Like WhatsApp, you need a dedicated phone number. It cannot be your primary Signal number.

  2. Register the number

    You'll need signal-cli installed on your server (or the OpenClaw image may include it). Register:

    signal-cli -u +1234567890 register
    signal-cli -u +1234567890 verify CODE
  3. Add to your agent
    clawctl channel add my-agent signal --number=+1234567890
  4. Approve contacts

    Signal defaults to allowlist mode. Add approved senders:

    clawctl channel allow my-agent signal +0987654321
Tip: Signal is the most privacy-focused option. Messages are end-to-end encrypted. The bot only sees messages from approved contacts.

Quick Reference

All the commands in one place.

# Add channels
clawctl channel add <agent> telegram --token=TOKEN
clawctl channel add <agent> discord --token=TOKEN
clawctl channel add <agent> slack --bot-token=xoxb-... --app-token=xapp-...
clawctl channel add <agent> whatsapp
clawctl channel add <agent> signal --number=+1234567890

# Or inline with create
clawctl create alice --telegram=TOKEN
clawctl create alice --discord=TOKEN

# Approve pairing
clawctl approve <agent> <channel> CODE

# Manage channels
clawctl channel status <agent>
clawctl channel security <agent>
clawctl channel send <agent> telegram --enable
clawctl channel allow <agent> telegram USER_ID
clawctl channel remove <agent> telegram

Security defaults applied on channel add:

  • DM policy: pairing (strangers must be approved with a code)
  • Outbound messages: OFF (agent cannot send unsolicited messages)
  • Reactions: ON (agent can react to messages)
  • Read-only info: ON (agent can see member/channel info)
  • Moderation/admin actions: OFF
