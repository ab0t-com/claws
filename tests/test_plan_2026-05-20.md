# Test Plan — clawctl Open-Source Readiness Audit (2026-05-20)

**Status.** Read-only ingestion of the codebase is complete (15.1k LOC source + tests, 232 Go tests, 8 tickets, prior reports). This plan exercises clawctl from a *cold-start operator's perspective* to confirm the README's promises and surface DX friction before we publish reports.

## Safety contract

The user runs **live agents on this host** under `~/.openclaw/` (groups `team/{john,lead,sarah}` and `team1/ben`). Every step in this plan obeys:

1. **Never** touch `~/.openclaw/`. Every clawctl invocation in this plan must set `OPENCLAW_ROOT=$(mktemp -d -t clawctl-dogfood-XXXXXX)` and export it in the same shell as the command. The smoke script enforces this.
2. **No `rm -rf`.** Scratch dirs are throwaway tempdirs; we leave them on disk for inspection.
3. **No `--purge`, no `policy enforce --restart`, no `proxy setup`** against any directory we haven't created ourselves in this session.
4. **No Docker mutations.** `CLAWCTL_SKIP_VALIDATE=1` is set in every smoke invocation so `docker compose config` is skipped and `portInUse()` short-circuits. We do not call `start`/`stop`/`restart`/`upgrade` — those would `docker compose up` real containers. Health probes are still issued (they will report `down` against the empty test root, which is the expected outcome).
5. **Verify the env before every batch.** Smoke script prints `OPENCLAW_ROOT` and refuses to run if it matches `$HOME/.openclaw`.

## What we're testing

We are not re-deriving correctness — the 232 existing Go tests already cover lifecycle, port allocation, config merge, override generation, role topology, policy, access, channels, storage, runtime adapter, file locking, and permissions. This plan instead exercises **operator-visible behavior** as a substitute for a real user and judges DX.

## Out of scope (this audit)

- Real Docker start/stop/restart (would touch live container runtime).
- Real channel tokens (require external accounts).
- S3/storage setup (requires AWS credentials).
- Proxy install (would mutate `/etc/caddy/`).
- Long-running flows like `dashboard` (interactive).
- `clawctl auth` OAuth flows (require a TTY).

These are covered by manual operator review elsewhere; this plan is enough to certify the *control-plane file behavior* and *DX surface* without taking the host through real lifecycle ops.
